# Problem ranking parameters (1–5 scale)

Score each parameter 1–5 for startup / side-project feasibility.

---

## 1. Problem Severity & Pain

| Parameter              | Purpose                                                                                                                                                                        |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **meaningful_problem** | Is it a real, relevant problem people would pay or care about? Could someone build a startup around it? 0 = no problem at all, 1 = small problem, 5 = relevant & real problem. |
| **pain_intensity**     | How painful is the problem? 1 = annoyance, 5 = mission-critical. Does it cause financial loss, emotional stress, or block core workflows?                                      |
| **frequency**          | How often does it occur? 1 = rare / episodic, 5 = daily / persistent.                                                                                                          |

---

## 2. Market Attractiveness

| Parameter              | Purpose                                                                                                                                         |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| **problem_size**       | How many people/companies have this problem? Choose between niche/global. Is it niche but high-value?                                           |
| **market_growth**      | Is demand increasing? Driven by regulation, tech shift, macro trends? 1 = stagnant or shrinking, 5 = strong growth.                             |
| **willingness_to_pay** | Do customers already pay for alternatives? Is budget allocated? Who owns the budget? 1 = small willingness to pay, 5 = big willingsness to pay. |

---

## 3. Customer Characteristics

| Parameter                   | Purpose                                                                                                                       |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **target_customer_clarity** | Can you clearly define the ideal customer? Is it easy to reach them? 1 = vague / hard to reach, 5 = clear ICP, easy to reach. |
| **problem_awareness**       | Do customers know they have the problem? 1 = must educate them, 5 = already aware (easy to sell).                             |

---

## 4. Competitive Landscape

| Parameter       | Purpose                                                                                                  |
| --------------- | -------------------------------------------------------------------------------------------------------- |
| **competition** | Is there white space? Can you 10x improve something? 1 = crowded / strong competitors, 5 = under-served. |

---

## 5. Solution Feasibility

| Parameter             | Purpose                                                                                                                                                                                                   |
| --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **software_solution** | How likely is software (any) to mitigate this problem? Can it be built with current tech? 1 = software will have a hard time solving this problem, 5 = software is the perfect way to solve this problem. |
| **ai_fit**            | How much does AI/ML meaningfully improve the solution (vs “just an app”)? 1 = AI/ML will have a hard time solving this problem, 5 = AI/ML is the perfect way to solve this problem.                       |
| **speed_to_mvp**      | How fast can you ship an MVP with a group of 3 developers? 1 = years, 5 = months.                                                                                                                         |

---

## 6. Unit Economics Potential

| Parameter              | Purpose                                                                                                                               |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **business_potential** | Revenue / startup potential. Clear monetization? Recurring? High retention expected? 1 = low business value, 5 = high business value. |

---

## 7. Timing & Trends

| Parameter          | Purpose                                                                                                                                              |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **time_relevancy** | Why is it important *now*? Regulatory changes, AI/tech shift, behavioral shift? 1 = not important now, 5 = highly relevant in the current landscape. |
